# 企业后台系统前端建设方案（Vue）

## 基础要求
1. 采用 Vue.js 搭建管理后台页面。
2. 覆盖后端已存在的核心组件能力：登录、角色、权限、批量授权。
3. 可连接后端接口与数据库（通过后端 API 完成数据存取）。

## 当前完成部分
1. 已在 `frontend/my-admin-demo/src/App.vue` 生成单页管理台原型。
2. 页面提供：
    - 登录获取 Token
    - 角色创建与分页读取
    - 权限创建与分页读取
    - 批量授权（角色-权限、用户-角色）
3. 使用浏览器 `fetch` 调后端接口，数据可写入数据库（通过后端 API 中转）。

## 还需要完成的工作（新增）
1. **工程化**：
    - API 模块化（`src/api/*`）
    - 鉴权拦截器（401 自动跳转登录）
    - 统一错误提示与日志上报
2. **权限前端化**：
    - 动态路由（按角色/权限显示菜单）
    - 按钮级权限指令（如 `v-permission`）
3. **UI 与可维护性**：
    - 引入组件库（Element Plus / Naive UI）
    - 表单校验、分页组件、弹窗二次确认
4. **联调与部署**：
    - 本地代理（Vite Proxy）
    - CORS 与 token 规范
    - Nginx 部署与环境变量配置
5. **质量保障**：
    - 单元测试 + E2E（Playwright/Cypress）
    - 接口契约（OpenAPI 自动生成 TS 类型）

## 推荐里程碑
- M1（1天）：模块化 API + 登录态持久化。
- M2（1天）：角色/权限管理页组件化。
- M3（1天）：权限路由 + 按钮鉴权。
- M4（0.5天）：联调回归 + 打包部署。
