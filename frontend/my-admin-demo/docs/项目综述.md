#### **3.1. 通信协议与数据格式**

#### **3.2. 开发环境跨域解决方案 (Vite 代理)**
在本地开发时，前端应用（如 `localhost:5173`）和后端 API（如 `localhost:8080`）运行在不同源上，会引发浏览器同源策略导致的跨域（CORS）问题 [[8]](https://comate.baidu.com/zh/page/9jet0z7ac6m)[[9]](https://apifox.com/apiskills/axios-request-encapsulation/) 。我们将利用 Vite 内置的开发服务器代理功能来解决此问题 [[1]](https://juejin.cn/post/7423307164046082059) 。

其原理是：前端发出的 API 请求被同源的 Vite 开发服务器拦截，然后由 Vite 服务器将请求转发给真实的后端服务器，从而巧妙绕开浏览器限制 [[8]](https://comate.baidu.com/zh/page/9jet0z7ac6m) 。

**`vite.config.js` 配置详解:**
```javascript
// vite.config.js
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';

export default defineConfig({
  plugins: [vue()],
  server: {
    open: true, // 启动服务时自动打开浏览器
    proxy: {
      // 规则：代理所有以 /api 开头的请求
      '/api': {
        target: 'http://localhost:8080', // 目标后端服务地址
        changeOrigin: true,              // 必须设置为 true，以支持跨域
        rewrite: (path) => path.replace(/^\/api/, ''), // 重写路径，去掉 /api 前缀
      },
      // 可配置多个代理规则，以应对微服务架构 [[10]](https://zxuqian.cn/vite-proxy-config)[[11]](https://medium.com/@eric_abell/simplifying-api-proxies-in-vite-a-guide-to-vite-config-js-a5cc3a091a2f)[[12]](https://developer.aliyun.com/article/1354889) 
      '/auth': {
        target: 'http://localhost:8081',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/auth/, ''),
      }
    }
  }
});
```
*   **`target`**: 指定代理的目标后端服务器地址 [[1]](https://juejin.cn/post/7423307164046082059)[[10]](https://zxuqian.cn/vite-proxy-config) 。
*   **`changeOrigin: true`**: 将请求头中的 `Host` 字段修改为目标地址的 `Host`，这对于需要验证 `Host` 头的后端服务至关重要 。
*   **`rewrite`**: 一个函数，用于重写请求路径。例如，前端请求 `/api/users`，经过重写后，后端收到的路径是 `/users` [[1]](https://juejin.cn/post/7423307164046082059)[[2]](https://www.cnblogs.com/lpkshuai/p/18222958)[[10]](https://zxuqian.cn/vite-proxy-config) 。

**重要提示**: 此代理配置仅在 **开发环境** (`vite dev`) 下生效。生产环境的跨域问题需要通过 Nginx 反向代理或后端配置 CORS 策略来解决 [[11]](https://medium.com/@eric_abell/simplifying-api-proxies-in-vite-a-guide-to-vite-config-js-a5cc3a091a2f)[[5]](https://comate.baidu.com/zh/page/haxg32brg2j) 。

**1. 创建 Axios 单例 (`src/utils/request.js`)**
我们创建一个具有通用配置（如 `baseURL` 和 `timeout`）的 Axios 实例 [[3]](https://comate.baidu.com/zh/page/q6kvda2iu69)[[4]](https://comate.baidu.com/zh/page/pdkoagp8wvk)[[13]](https://blog.csdn.net/weixin_48179599/article/details/107028209)[[14]](https://comate.baidu.com/zh/page/oikc5l3ardh)[[15]](https://blog.csdn.net/clli_Chain/article/details/128416636) 。`baseURL` 将通过环境变量动态设置，以适应不同环境（开发、测试、生产 [[9]](https://apifox.com/apiskills/axios-request-encapsulation/)[[12]](https://developer.aliyun.com/article/1354889)）。
```javascript

import { ElMessage, ElMessageBox } from 'element-plus';
import { useUserStore } from '@/stores/user'; // 引入用户 store

const service = axios.create({
```

**2. 配置请求拦截器 (Request Interceptor)**
利用请求拦截器，为每个需要认证的 API 请求动态添加 JWT Token [[16]](https://juejin.cn/post/7307857934923137075)[[6]](https://axios.nodejs.cn/docs/instance)[[17]](https://axios-http.com/zh/docs/config_defaults)[[18]](https://axios.org.cn/docs/instance) 。
```javascript
// src/utils/request.js (续)
service.interceptors.request.use(
  (config) => {
    const userStore = useUserStore();
    if (userStore.token) {
      // 'Bearer ' 是 JWT 的标准前缀 [[2]](https://www.cnblogs.com/lpkshuai/p/18222958) 
      config.headers['Authorization'] = `Bearer ${userStore.token}`;
    }
    return config;
  },
  (error) => {
    console.error('Request Interceptor Error:', error);
    return Promise.reject(error);
  }
);
```

**3. 配置响应拦截器 (Response Interceptor)**
响应拦截器是全局处理后端响应的关键，用于统一处理成功数据、业务错误和 HTTP 状态码错误 [[6]](https://axios.nodejs.cn/docs/instance)[[7]](https://medium.com/@tessintaiwan/how-to-automatically-attach-jwt-token-to-axios-requests-3ab9741e3fb8) 。
```javascript
// src/utils/request.js (续)
service.interceptors.response.use(
  (response) => {
    const res = response.data;
    // 假设后端通过自定义 code 字段表示业务状态，例如 20000 为成功
    if (res.code !== 20000) {
      ElMessage({ message: res.message || 'Error', type: 'error' });

      // 特定业务错误码处理，例如 Token 过期
      if ([50008, 50012, 50014].includes(res.code)) {
        handleTokenExpired();
      }
      return Promise.reject(new Error(res.message || 'Error'));
    } else {
      // 业务成功，直接返回数据部分
      return res.data;
    }
  },
  (error) => {
    // HTTP 状态码错误处理
    if (error.response) {
      switch (error.response.status) {
        case 401: // Unauthorized: Token 无效或过期
          handleTokenExpired();
          break;
        case 403: // Forbidden: 用户无权限
          ElMessage.error('您没有权限执行此操作。');
          break;
        case 500: // Internal Server Error
          ElMessage.error('服务器发生错误，请稍后重试。');
          break;
        default:
          ElMessage.error(`请求错误：${error.response.status}`);
      }
    } else if (error.message.includes('timeout')) {
      ElMessage.error('请求超时，请检查您的网络连接。');
    } else {
      ElMessage.error('网络连接异常，请检查您的网络设置。');
    }
    return Promise.reject(error);
  }
);

// 统一处理 Token 过期/失效的函数
function handleTokenExpired() {
  ElMessageBox.confirm('您的登录状态已失效，请重新登录', '确认登出', {
    confirmButtonText: '重新登录',
    cancelButtonText: '取消',
    type: 'warning',
  }).then(() => {
    const userStore = useUserStore();
    userStore.logout().then(() => {
      // 重定向到登录页，并带上当前页面路径以便登录后跳回
      router.push(`/login?redirect=${router.currentRoute.value.fullPath}`);
    });
  });
}

export default service;
```

### **6. 精细化权限控制方案**
这是企业级应用的核心，我们将实现页面级和元素级两个层面的权限控制。核心思想是：前端定义好所有路由，用户登录后根据后端返回的权限标识，动态计算出可访问的路由，然后通过 `router.addRoute()` 添加到路由实例中 [[1]](https://juejin.cn/post/7423307164046082059)[[2]](https://www.cnblogs.com/lpkshuai/p/18222958) 。

#### **6.1. 状态管理 (Pinia)**
我们将使用三个 Store 协同工作：`user` store 负责用户认证和权限标识，`permission` store 负责生成动态路由，`layout` store 负责管理布局状态 [[1]](https://juejin.cn/post/7423307164046082059) 。

*   **User Store (`stores/user.js`):** 管理用户 Token、基本信息和权限标识（`permissionCodes`）。
    ```javascript
    // src/stores/user.js
    import { defineStore } from 'pinia';
    import { loginApi, getUserInfoApi } from '@/api/user';
    import { usePermissionStore } from './permission';

    export const useUserStore = defineStore('user', {
      state: () => ({
        token: localStorage.getItem('token') || '',
        userInfo: JSON.parse(localStorage.getItem('userInfo')) || {},
        // 后端返回的权限标识码数组，例如 ['system:user:list', 'system:user:add']
        permissionCodes: JSON.parse(localStorage.getItem('permissionCodes')) || []
      }),
      getters: {
        hasPermission: (state) => (code) => state.permissionCodes.includes(code),
      },
      actions: {
        async login(credentials) {
          const { token } = await loginApi(credentials);
          this.token = token;
          localStorage.setItem('token', token);
          // 登录后立即获取用户信息和权限
          await this.getUserInfo();
        },
        async getUserInfo() {
          const { userInfo, permissionCodes } = await getUserInfoApi();
          this.userInfo = userInfo;
          this.permissionCodes = permissionCodes;
          localStorage.setItem('userInfo', JSON.stringify(userInfo));
          localStorage.setItem('permissionCodes', JSON.stringify(permissionCodes));
          return permissionCodes; // 返回权限码供路由守卫使用
        },
        async logout() {
          // 可选：调用后端登出接口
          this.token = '';
          this.userInfo = {};
          this.permissionCodes = [];
          localStorage.clear();
          // 重置动态路由
          const permissionStore = usePermissionStore();
          permissionStore.resetRoutes();
        }
      }
    });
    ```
*   **Permission Store (`stores/permission.js`):** 这是权限管理的核心，负责根据权限标识过滤异步路由表 。
    ```javascript
    // stores/permission.js
    import { defineStore } from 'pinia';
    import { asyncRoutes, constantRoutes } from '@/router';

    /**
     * 递归过滤异步路由表
     * @param routes asyncRoutes
     * @param codes 用户拥有的权限码
     */
    function filterAsyncRoutes(routes, codes) {
      const res = [];
      routes.forEach(route => {
        const tmp = { ...route };
        // 检查路由元信息中是否定义了权限码
        if (tmp.meta && tmp.meta.permission) {
          // 如果用户权限码包含路由所需的权限码，则该路由可访问
          if (codes.includes(tmp.meta.permission)) {
            if (tmp.children) {
              tmp.children = filterAsyncRoutes(tmp.children, codes);
            }
            res.push(tmp);
          }
        } else {
          // 如果路由没有定义权限码，则默认所有用户都可访问
          if (tmp.children) {
            tmp.children = filterAsyncRoutes(tmp.children, codes);
          }
          res.push(tmp);
        }
      });
      return res;
    }

    export const usePermissionStore = defineStore('permission', {
      state: () => ({
        routes: [], // 存储当前用户所有可访问的完整路由
        addRoutes: [], // 存储动态添加的路由
      }),
      actions: {
        generateRoutes(codes) {
          return new Promise(resolve => {
            let accessedRoutes;
            // 如果是超级管理员，拥有所有路由
            if (codes.includes('*:*:*')) {
              accessedRoutes = asyncRoutes || [];
            } else {
              accessedRoutes = filterAsyncRoutes(asyncRoutes, codes);
            }
            this.addRoutes = accessedRoutes;
            this.routes = constantRoutes.concat(accessedRoutes);
            resolve(accessedRoutes);
          });
        },
        resetRoutes() {
            this.routes = [];
            this.addRoutes = [];
        }
      },
    });
    ```
*   **Layout Store (`stores/layout.js`):** 一个简单的 Store，用于控制侧边栏的展开和折叠 。
    ```javascript
    // stores/layout.js
    import { defineStore } from 'pinia';

    export const useLayoutStore = defineStore('layout', {
      state: () => ({
        isSidebarCollapsed: false, // 侧边栏是否折叠
      }),
      actions: {
        toggleSidebar() {
          this.isSidebarCollapsed = !this.isSidebarCollapsed;
        },
      },
    });
    ```

#### **6.2. 路由控制 (页面级权限)**
我们将路由分为**常量路由**（所有用户可访问）和**异步路由**（需权限验证 [[1]](https://juejin.cn/post/7423307164046082059)[[2]](https://www.cnblogs.com/lpkshuai/p/18222958)）。

1.  **定义路由 (`router/index.js`):** 在路由元信息 `meta` 中添加 `permission` 字段 [[2]](https://www.cnblogs.com/lpkshuai/p/18222958) 。
    ```javascript
    // router/index.js
    import { createRouter, createWebHistory } from 'vue-router';
    import Layout from '@/layouts/index.vue';

    // 公共路由
    export const constantRoutes = [
      { path: '/login', component: () => import('@/views/LoginView.vue'), hidden: true },
      { path: '/404', component: () => import('@/views/error/404.vue'), hidden: true },
      { 
        path: '/', 
        component: Layout, 
        redirect: '/dashboard',
        children: [{ path: 'dashboard', name: 'Dashboard', component: () => import('@/views/Dashboard.vue'), meta: { title: '首页' } }]
      }
    ];

    // 异步路由（需要权限）
    export const asyncRoutes = [
      {
        path: '/system',
        component: Layout,
        redirect: '/system/user',
        meta: { title: '系统管理', permission: 'system' }, // 父级权限标识
        children: [
          {
            path: 'user',
            name: 'UserManagement',
            component: () => import('@/views/system/UserManagement.vue'),
            meta: { title: '用户管理', permission: 'system:user:list' } // 页面权限标识
          },
        ]
      },
      // 404页面必须最后加载
      { path: '/:pathMatch(.*)*', redirect: '/404', hidden: true } 
    ];

    const router = createRouter({
        history: createWebHistory(),
        routes: constantRoutes,
    });

    export default router;
    ```
2.  **创建导航守卫 (`permission.js`):** 这是权限控制的核心，在每次路由跳转前进行检查 [[1]](https://juejin.cn/post/7423307164046082059)[[2]](https://www.cnblogs.com/lpkshuai/p/18222958) 。
    ```javascript
    // src/permission.js
    import router from './router';
    import { useUserStore } from './stores/user';
    import { usePermissionStore } from './stores/permission';

    const whiteList = ['/login']; // 路由白名单

    router.beforeEach(async (to, from, next) => {
      const userStore = useUserStore();
      const hasToken = userStore.token;

      if (hasToken) {
        if (to.path === '/login') {
          next({ path: '/' });
        } else {
          const hasPermissions = userStore.permissionCodes && userStore.permissionCodes.length > 0;
          if (hasPermissions) {
            next();
          } else {
            try {
              // 刷新页面后，重新获取用户信息和权限
              const codes = await userStore.getUserInfo();
              const permissionStore = usePermissionStore();
              const accessRoutes = await permissionStore.generateRoutes(codes);
              
              accessRoutes.forEach(route => router.addRoute(route));
              
              // hack方法，确保addRoute已完成
              next({ ...to, replace: true });
            } catch (error) {
              // 获取权限失败，可能是 token 失效，登出并重定向到登录页
              await userStore.logout();
              next(`/login?redirect=${to.path}`);
            }
          }
        }
      } else {
        if (whiteList.includes(to.path)) {
          next();
        } else {
          next(`/login?redirect=${to.path}`);
        }
      }
    });
    ```
    **别忘了在 `main.js` 中引入它:** `import './permission';` [[1]](https://juejin.cn/post/7423307164046082059) 。

#### **6.3. 元素级权限控制**
推荐使用**自定义指令 `v-permission`**，它使模板更具声明性 [[1]](https://juejin.cn/post/7423307164046082059)[[2]](https://www.cnblogs.com/lpkshuai/p/18222958) 。

1.  **注册全局指令 (`main.js`):**
    ```javascript
    // src/main.js
    // ...
    app.directive('permission', {
      mounted(el, binding) {
        const { value } = binding;
        const userStore = useUserStore();
        if (value && Array.isArray(value) && value.length > 0) {
          const hasPermission = value.some(p => userStore.hasPermission(p));
          if (!hasPermission) {
            el.parentNode && el.parentNode.removeChild(el);
          }
        }
      }
    });
    ```
2.  **在组件中使用:**
    ```vue
    <el-button type="primary" v-permission="['system:user:add']">新增用户</el-button>
    ```

### **7. 其他开发注意事项**

*   **代码风格与规范:**
    *   项目已通过 `create-vue` 集成 **ESLint** 和 **Prettier**。所有开发者应遵循配置，并开启“保存时自动格式化”。

*   **环境配置管理:**
    *   Vite 通过内置的 `dotenv` 支持，提供了一套强大且易于管理的环境变量机制 [[7]](https://medium.com/@tessintaiwan/how-to-automatically-attach-jwt-token-to-axios-requests-3ab9741e3fb8)[[8]](https://comate.baidu.com/zh/page/9jet0z7ac6m) 。
    *   **文件体系**: 在项目根目录创建 `.env` 系列文件来管理不同环境的配置 [[1]](https://juejin.cn/post/7423307164046082059) 。
        *   `.env.development`: 用于开发环境 (`npm run dev`)，Vite 会进入 `development` 模式 。
        *   `.env.production`: 用于生产构建 (`npm run build`)，Vite 会进入 `production` 模式 [[2]](https://www.cnblogs.com/lpkshuai/p/18222958) 。
        *   `.env.[mode]`: 用于自定义模式，例如为测试环境创建一个 `.env.staging` 文件，并通过 `npm run build --mode staging` 命令来使用它 [[11]](https://medium.com/@eric_abell/simplifying-api-proxies-in-vite-a-guide-to-vite-config-js-a5cc3a091a2f) 。
    *   **变量规则**: 为了安全，只有以 `VITE_` 为前缀的变量才会暴露给客户端代码 [[10]](https://zxuqian.cn/vite-proxy-config)[[8]](https://comate.baidu.com/zh/page/9jet0z7ac6m)[[9]](https://apifox.com/apiskills/axios-request-encapsulation/) 。
        ```env
        # .env.development
        VITE_API_BASE_URL=/api
        
        # .env.production
        VITE_API_BASE_URL=https://prod.example.com/api
        ```
    *   **代码中使用**: 在代码中，通过 `import.meta.env.VITE_API_BASE_URL` 来访问这些变量，如在 Axios 的 `baseURL` 配置中 [[2]](https://www.cnblogs.com/lpkshuai/p/18222958)[[12]](https://developer.aliyun.com/article/1354889) 。

*   **生产环境部署考量:**
    *   开发环境中的 Vite 代理 (`server.proxy`) 无法用于生产环境 [[11]](https://medium.com/@eric_abell/simplifying-api-proxies-in-vite-a-guide-to-vite-config-js-a5cc3a091a2f)[[5]](https://comate.baidu.com/zh/page/haxg32brg2j) 。
    *   **推荐方案：Nginx 静态托管与反向代理**。这是最经典和推荐的生产部署方案 。
        *   **静态资源托管**: Nginx 直接托管 Vite 构建出的 `dist` 目录下的静态文件 [[5]](https://comate.baidu.com/zh/page/haxg32brg2j) 。
        *   **解决 History 模式 404**: 配置 `try_files $uri $uri/ /index.html;` 来确保在刷新页面时，请求能被正确地交由 Vue Router 处理 [[5]](https://comate.baidu.com/zh/page/haxg32brg2j)[[10]](https://zxuqian.cn/vite-proxy-config)[[13]](https://blog.csdn.net/weixin_48179599/article/details/107028209) 。
        *   **API 反向代理**: 配置 `location /api/` 将所有 API 请求转发到后端的 Spring Boot 服务，从而解决生产环境的跨域问题 [[9]](https://apifox.com/apiskills/axios-request-encapsulation/)[[11]](https://medium.com/@eric_abell/simplifying-api-proxies-in-vite-a-guide-to-vite-config-js-a5cc3a091a2f) 。

*   **CSS 方案:**
    *   强烈推荐使用 Vue 单文件组件 (SFC) 中的 **Scoped CSS** (`<style scoped>`)。
    *   对于需要覆盖子组件样式的场景，可以使用深度选择器 `:deep()`。

```vue
<template>
  <div v-if="!item.hidden">
    <template v-if="hasOneShowingChild(item.children, item) && (!onlyOneChild.children || onlyOneChild.noShowingChildren)">
      <el-menu-item :index="resolvePath(onlyOneChild.path)">
        <template #title>{{ onlyOneChild.meta.title }}</template>
      </el-menu-item>
    </template>
    <el-sub-menu v-else :index="resolvePath(item.path)">
      <template #title>
        <span>{{ item.meta.title }}</span>
      </template>
      <SidebarItem v-for="child in item.children" :key="child.path" :item="child" :base-path="resolvePath(item.path)" />
    </el-sub-menu>
  </div>
</template>
<script setup>
import { ref } from 'vue';
import path from 'path-browserify';

const props = defineProps({
  item: { type: Object, required: true },
  basePath: { type: String, default: '' },
});

const onlyOneChild = ref(null);

function hasOneShowingChild(children = [], parent) {
  const showingChildren = children.filter(item => {
    if (item.hidden) return false;
    onlyOneChild.value = item;
    return true;
  });
  if (showingChildren.length === 1) return true;
  if (showingChildren.length === 0) {
    onlyOneChild.value = { ...parent, path: '', noShowingChildren: true };
    return true;
  }
  return false;
}

function resolvePath(routePath) {
  return path.resolve(props.basePath, routePath);
}
</script>
```

```vue
  pageSize: 10,

    loading.value = false;

}

  queryParams.sortField = prop;
  queryParams.sortOrder = order;
  queryParams.pageNum = 1; 
  getList();
}

function handleSizeChange(newSize) {
  queryParams.pageSize = newSize;
  queryParams.pageNum = 1;
  getList();
}

  queryParams.pageNum = newPage;
  getList();
}

  queryParams.pageNum = 1;
  getList();
}

  queryParams.sortField = '';
  queryParams.sortOrder = '';
  queryParams.pageNum = 1;
  getList();
}

});
```

### **10. 总结**
本开发文档在原有的基础上，深度整合了企业级应用中至关重要的**前后端交互方案**和**精细化权限控制方案**，并提供了更贴近真实业务场景的代码实现 [[1]](https://juejin.cn/post/7423307164046082059)[[2]](https://www.cnblogs.com/lpkshuai/p/18222958) 。

在前后端集成方面，我们定义了一套健壮且可维护的策略。开发阶段，通过配置 **Vite 的 `server.proxy`**，有效解决了本地开发的跨域问题 [[1]](https://juejin.cn/post/7423307164046082059)[[8]](https://comate.baidu.com/zh/page/9jet0z7ac6m) 。对于所有环境，我们通过**封装 Axios** 创建了一个统一的 API 请求层 [[3]](https://comate.baidu.com/zh/page/q6kvda2iu69)[[5]](https://comate.baidu.com/zh/page/haxg32brg2j) 。该封装利用**请求拦截器**自动附加认证 Token [[16]](https://juejin.cn/post/7307857934923137075)[[17]](https://axios-http.com/zh/docs/config_defaults) ，并利用**响应拦截器**全局处理业务成功、业务失败以及 HTTP 状态码错误（如 401 自动登出、403 权限提示等），极大地提升了代码质量和开发效率 [[6]](https://axios.nodejs.cn/docs/instance)[[7]](https://medium.com/@tessintaiwan/how-to-automatically-attach-jwt-token-to-axios-requests-3ab9741e3fb8) 。

在权限控制方面，方案的核心是**基于权限标识 (Permission Codes) 的动态路由和元素级控制** [[2]](https://www.cnblogs.com/lpkshuai/p/18222958) 。我们详细阐述了如何通过 `user`、`permission`、`layout` 三个 Pinia Store 协同工作，实现职责分离 [[1]](https://juejin.cn/post/7423307164046082059) 。利用 Vue Router 的导航守卫，我们实现了**页面级权限**，确保用户登录后，能根据其权限动态生成可访问的路由和侧边栏菜单 。同时，通过自定义指令 `v-permission`，我们以一种优雅且声明式的方式实现了**元素级权限**，动态显隐页面内的操作按钮 。

在核心功能实现上，本文档提供了一套完整的**动态布局方案**，包括主布局、头部、以及支持多级嵌套的递归侧边栏菜单，为应用提供了专业的 UI 骨架 。更重要的是，我们用一个**与后端深度交互的数据表格**示例，取代了简单的纯前端展示。该示例详细演示了如何统一管理查询参数，并响应用户操作（如点击排序、切换页面、条件筛选）来动态请求服务端数据，完整地展现了企业级后台管理系统中**服务端分页、排序、筛选**的最佳实践 [[1]](https://juejin.cn/post/7423307164046082059)[[2]](https://www.cnblogs.com/lpkshuai/p/18222958) 。

此外，文档还重点强调了**多环境配置管理**的重要性，详细阐述了如何利用 Vite 的 `.env` 文件体系和 `VITE_` 前缀变量来管理开发、测试和生产环境的差异化配置 [[1]](https://juejin.cn/post/7423307164046082059)[[10]](https://zxuqian.cn/vite-proxy-config)[[8]](https://comate.baidu.com/zh/page/9jet0z7ac6m) 。同时，简要说明了生产环境部署的最佳实践，即采用 **Nginx 反向代理**，为项目从开发到上线的整个生命周期提供了清晰的指导 [[5]](https://comate.baidu.com/zh/page/haxg32brg2j)[[11]](https://medium.com/@eric_abell/simplifying-api-proxies-in-vite-a-guide-to-vite-config-js-a5cc3a091a2f) 。

整个架构职责清晰，在满足不使用 TypeScript 的前提下，构建了一个安全、可扩展、易于维护的现代化前端应用基础，能够高效地支持复杂业务需求的开发与迭代。
