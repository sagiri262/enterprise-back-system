# 企业后台数据库设计规范

## 1. 命名规范

- 系统权限相关表使用 `sys_` 前缀，例如 `sys_user`、`sys_role`。
- 业务模块表使用 `biz_` 前缀，例如 `biz_department`、`biz_employee`。
- 表名、字段名统一使用小写下划线风格。
- 主键统一命名为 `id`，类型为 `BIGINT AUTO_INCREMENT`。
- 唯一索引命名为 `uk_表名_字段`，普通索引命名为 `idx_表名_字段`。

## 2. 通用字段

业务表建议至少包含：

```sql
id BIGINT NOT NULL AUTO_INCREMENT
create_time DATETIME DEFAULT NULL
update_time DATETIME DEFAULT NULL
```

有启停语义的表使用：

```sql
status TINYINT NOT NULL DEFAULT 1 COMMENT '状态 1启用 0禁用'
```

涉及金额的字段使用 `DECIMAL(12,2)`，禁止使用 `FLOAT/DOUBLE`。

## 3. 当前业务表

| 表名 | 说明 | 主要用途 |
|---|---|---|
| `biz_department` | 部门表 | 组织架构、部门负责人 |
| `biz_employee` | 员工表 | 在线员工、联系权限、组织人员 |
| `biz_notice` | 消息通知表 | 消息中心、系统提醒 |
| `biz_oa_process` | OA流程应用表 | 工作台应用入口 |
| `biz_expense` | 费用记录表 | 费用中心、报销统计 |

## 4. 关系设计

- `biz_employee.department_id` 关联 `biz_department.id`。
- `biz_department.manager_employee_id` 表示部门负责人。
- `biz_oa_process.owner_employee_id` 表示流程负责人。
- `biz_expense.applicant_employee_id` 表示费用申请人。
- `biz_expense.department_id` 表示费用归属部门。

当前版本只对员工所属部门建立强外键，其他负责人字段保留弱关联，方便初始化和业务演进。

## 5. 权限规范

新增业务接口必须同步写入 `sys_permission`，命名格式：

```text
模块:动作
```

示例：

```text
department:list
department:create
employee:update
expense:delete
```

Controller 必须使用 `@PreAuthorize` 做方法级权限校验。写操作必须加 `@AuditLog`。

## 6. 数据安全规范

- 密码只允许存储 BCrypt 哈希，禁止明文入库。
- 登录、授权、员工、费用等敏感接口必须经过 JWT 认证。
- 写操作必须记录审计日志。
- 审计日志中的 password、token 等字段必须脱敏。
- 生产环境必须使用 HTTPS。
- 员工联系方式属于敏感数据，后续应按 `contact_level` 做数据级权限过滤。

## 7. 初始化脚本

- 基础系统表：`src/main/resources/sql/init.sql`
- 业务表和样例数据：`src/main/resources/sql/business_init.sql`

首次初始化顺序：

```bash
mysql -uroot -p < src/main/resources/sql/init.sql
mysql -uroot -p < src/main/resources/sql/business_init.sql
```
