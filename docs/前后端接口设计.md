# 前后端接口设计

## 1. 认证

| 功能 | 方法 | 路径 | 权限 |
|---|---|---|---|
| 登录 | POST | `/api/auth/login` | 公开 |
| 测试注册 | POST | `/api/auth/register-test` | 公开，生产应关闭 |

登录成功返回 JWT，前端后续请求使用：

```http
Authorization: Bearer <token>
```

## 2. 门户首页

| 功能 | 方法 | 路径 | 权限 |
|---|---|---|---|
| 后台首页聚合数据 | GET | `/api/portal/overview` | 已登录 |

返回内容包括：

- `metrics`：首页统计卡片
- `notices`：消息中心
- `contactRules`：联系权限规则
- `orgGroups`：组织架构
- `workApps`：OA 工作台应用
- `financeItems`：费用中心统计
- `financeFlows`：费用流程说明
- `rightLinks`：右侧快捷入口

## 7. 安全关键点

- 前端统一使用 HTTPS 部署，开发环境通过 Vite Proxy 转发 `/api`。
- 后端使用 JWT 过滤器识别用户身份。
- 所有非公开接口要求登录。
- 管理类接口使用 `@PreAuthorize("hasAuthority('xxx')")`。
- 写操作加 `@AuditLog`，记录用户、路径、参数、耗时和成功/失败状态。
- 前端不能把 token 写入 URL，只能放入请求头。
- 后端审计参数必须脱敏 password/token。
